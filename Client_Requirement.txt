A client wants a simple command-line application to keep track of daily expenses.

The application should help them record expenses, organize them by category, update mistakes, delete expenses, search expenses, and generate useful reports.

The application should save all data, so when the program is restarted, nothing is lost.

Functional Requirements:

Main Menu:

============================
 Expense Tracker
============================

1. Add Expense
2. View All Expenses
3. Search Expense
4. Update Expense
5. Delete Expense
6. Monthly Summary
7. Category Report
8. Exit

Expense Information

Each expense should contain:

- Expense ID
- Title
- Amount
- Category
- Date

Example:

ID: e1023

Title: Pizza

Amount: 450

Category: Food

Date: 27-06-2026

Validation Rules
Title cannot be empty.
Amount must be greater than 0.
Category cannot be empty.
Date must be valid (you may initially accept any non-empty string and improve validation later).
